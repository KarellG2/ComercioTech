4.- Requisitos 
	4.1.- Gestion Clientes - Prioridad: Alta
Descripción:
El sistema debe permitir crear, actualizar, consultar y eliminar perfiles de clientes
	
	4.2.- Gestion Productos - Prioridad: Alta
Descripción:
CRUD del catalogo con esquema flexible para variabilidad de categorías

	4.3.- Gestion Pedidos - Prioridad: Alta
Descripción:
Registro pedidos asociados a tablas cliente - productos, gestionar estados || pendiente - procesado - ¿enviado? ¿entregado? - cancelado || y datetime para cada uno

	4.4.- Historial - Prioridad: Alta
Descripción:
Almacenamiento y consultas del historial de compras por cliente y rango de fecha

	4.5.- Búsquedas y filtros - Prioridad: Media
Descripción:
Búsquedas por campos (Nombre - Categoría - Precio - Estado - Fecha), Ordenamiento por criterio

	4.6 Roles y mínimo privilegio necesario - Prioridad: Media
Descripción:
Implementación de inicio de sesión diferido empleado - administrador

	4.7 Seguridad de la Información - Prioridad: Alta
Descripción:
Implementar medidas de seguridad en los datos almacenados, hashing y encriptado para los datos en transito y en reposo


Justificación de Gestor de Base de Datos:

Los motivos de la selección de MongoDB como sistema gestor de la base de datos NoSQL son las siguientes:


1. Modelado Flexible:MongoDB permite representar clientes, productos y pedidos como documentos JSON - BSON con estructuramiento flexible 

2. Escalabilidad nativa: MongoDB distribuye automáticamente las colecciones entre multiples nodos, lo que satisface las proyecciones de crecimiento para una empresa como esta

3. Replica sets: Garantía de alta disponibilidad sin intervención manual ante caída del nodo primario

4. Seguridad: Autenticación y cifrado en transito y reposo con auditorias configurables
